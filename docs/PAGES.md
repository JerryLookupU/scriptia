# 页面需求细化

> 说明：所有页面默认包含“中间 Chat + 右侧双区 + 底部固定输入框”，并遵循“一切皆卡片”。

## 1. 首页
页面定位：单一主入口 + 意图路由
核心目标：快速开始对话、自动识别意图并引导到正确模块
主要区域：
- 中间 Chat：初始状态居中输入；发送后输入框下沉固定
- 右侧双区：暂存区 + 卡片库（全局可见）
- 左侧侧边栏：模块入口
主要能力：
- 输入自然语言，AI 识别意图并提示跳转到目标页
- 输出最小必要卡片（避免噪音）
- 历史记录：缩略会话列表 → 选择跳转
关键交互：
- “随机生成卡片”按钮：点击后在对话中弹出卡片
- 对话中卡片支持：引用/保存/折叠/查看/生成配图

## 2. 灵感
页面定位：抽卡与组合的轻量舞台
核心目标：快速获得灵感卡片并组合
主要区域：
- 中间 Chat：输入需求 → 返回灵感卡片
- 工作区：卡片排布与组合（可拖拽）
主要能力：
- 生成灵感卡片（随机或按类型）
- 卡片拖拽组合形成新的结果卡
- 结果卡一键转为项目卡片
关键交互：
- 生成卡片后自动进入工作区
- 对话驱动“组合/替换/重写”

## 3. 项目
页面定位：从需求到脚本产出的主场
核心目标：选题发散、脚本生成、版本对比
主要区域：
- 中间 Chat：输入需求/新闻 → 生成项目卡
- 工作区：项目卡片池 + 脚本草稿区 + 版本对比区
主要能力：
- 项目内生成的卡片默认是“项目卡片”
- 支持新闻选题发散：输入新闻 → 新闻卡片 → 选题列表条目 → 选题卡
- 支持脚本结构化产出（标题/开场/转折/结尾）
关键交互：
- 选题条目一键生成项目卡并拖拽到草稿区
- 结果卡可收藏为全局卡（需管理员权限）

## 4. AI 角色聊天室
页面定位：角色驱动对话的生成区
核心目标：用角色卡驱动对话生成
主要区域：
- 中间 Chat：用户输入角色对话指令
- 角色区：从卡片库拖拽角色卡
主要能力：
- 手动调度：用户指定角色发言
- 自动调度：AI 控制发言顺序
关键交互：
- 角色卡拖入角色区后，自动生成“角色上下文卡”
- 对话结果生成可保存为项目卡

## 5. 卡片库
页面定位：全局卡片的管理中心
核心目标：类型管理、模板管理、只读全局卡
主要区域：
- 右侧卡片库：类型视图、卡片列表
- 中间 Chat：输入文本生成卡片
主要能力：
- 类型管理：字段、提示词模板配置
- 全局卡只读；复制后成为项目卡
关键交互：
- 卡片复制 → 进入项目卡片池
- 管理员可将项目卡升级为全局卡

## 6. AI 工具箱
页面定位：媒体生成能力扩展
核心目标：生成图片/视频作为卡片配图
主要区域：
- 中间 Chat：输入生成需求
- 结果区：媒体结果卡片
主要能力：
- 生成图片/视频并绑定到卡片
关键交互：
- “生成配图”弹窗 → 生成后自动挂到卡片

## 7. 资料库
页面定位：资料导入与归档
核心目标：导入、切分、卡片化
主要区域：
- 中间 Chat：输入资料处理需求
- 工作区：资料卡片与目录结构
主要能力：
- 导入文件并切分为卡片
- 目录设计与归档
关键交互：
- 资料卡可一键转为项目卡或全局卡
