# 详细知识库（Admin 可见）

> 说明：本文件为“卡片知识库”规范与样例库，默认仅管理员可见/可编辑。

## 1. 知识库定位
- 目标：沉淀高质量、可复用的创作素材与结构化知识
- 形态：以“全局卡片”为核心的只读库，支持复制为项目卡
- 受众：管理员维护、团队成员引用

## 2. 权限与可见性
- 管理员：可创建/编辑/发布全局卡片与类型模板
- 普通用户：仅可浏览全局卡片；复制后生成项目卡片可编辑
- 全局卡片来源：管理员创建或从项目卡精选入库

## 3. 卡片类型规范

### 3.1 IP 角色卡
- 字段：角色名、身份标签、口吻/语气、关键动机、禁忌点、典型台词
- 用途：角色一致性、对话生成、剧情驱动
- 示例：
  - 角色名：路北
  - 身份标签：互联网产品经理/理性派
  - 口吻：冷静、简洁、条理清晰
  - 动机：用数据证明决策
  - 禁忌：情绪化表述、夸张承诺
  - 典型台词："先定义问题，再谈方案"

### 3.2 场景卡
- 字段：场景名称、时间/地点、环境氛围、冲突触发点、可用道具
- 用途：构建画面与语境
- 示例：
  - 场景名称：夜班办公室
  - 时间/地点：凌晨/开放工位
  - 氛围：疲惫、压迫
  - 冲突触发点：上线前 Bug
  - 道具：咖啡杯、倒计时屏

### 3.3 剧情卡
- 字段：剧情目标、冲突、转折点、结果
- 用途：驱动故事走向
- 示例：
  - 剧情目标：争取延期
  - 冲突：老板要求按期上线
  - 转折点：数据证明风险
  - 结果：批准延期一周

### 3.4 梗指南卡
- 字段：梗名称、使用场景、情绪强度、注意事项
- 用途：提升传播性与记忆点
- 示例：
  - 梗名称：反向夸奖
  - 使用场景：吐槽式夸赞
  - 情绪强度：中等
  - 注意事项：避免人身攻击

### 3.5 自定义类型卡
- 字段：类型名、字段列表、提示词模板、示例
- 用途：适配不同团队或行业需求

## 4. 新闻选题发散卡规范
- 卡片类型：新闻卡
- 字段：新闻标题、原始内容、关键词、主叙事、风险提示
- 选题列表：条目列表（每条可一键生成选题卡）
- 选题条目字段：角度、平台适配、标题建议、传播点

## 5. 脚本产出卡规范
- 卡片类型：脚本结果卡
- 字段：标题、结构（开场/冲突/转折/结尾）、目标平台、时长、BGM/镜头建议
- 版本：支持版本号与差异描述

## 6. 卡片质量规则（管理员）
- 草稿：信息不完整，仅供内部参考
- 可用：字段齐全，可直接复用
- 优秀：结构清晰，适配明确平台与场景

## 7. 提示词模板（管理员）
- 模板结构：角色/场景/剧情/梗/限制条件/输出格式
- 约束：避免敏感、避免虚假承诺、避免过度夸张

## 8. 样例库（管理员维护）
- 角色卡：3-5 个高质量样例
- 场景卡：3-5 个高质量样例
- 剧情卡：3-5 个高质量样例
- 梗指南卡：3-5 个高质量样例
- 新闻卡：1-2 个完整新闻发散样例

## 9. 变更与审核
- 任何全局卡片变更需管理员审核
- 建议增加变更日志（时间/人/原因）
